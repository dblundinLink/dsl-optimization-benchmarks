# DSL Specification: Binary Neural Network STE Training

## 1. OVERVIEW

This DSL defines training procedures for 2-layer MLPs with binary weights {-1,+1} and binary activations using the straight-through estimator (STE). It separates weight quantization from gradient estimation, enabling systematic exploration of STE variants (identity vs clipped), learning rate schedules, and optimizer configurations. The DSL compiles to a Python `train` function that executes 200 update steps on 500 MNIST examples and returns validation classification error.

## 2. OPERATORS

| Name | Args | Semantics | Example |
|------|------|-----------|---------|
| `LAYER` | `(in, out, init)` | Define dense layer with dimensions and weight init (xavier\|kaiming\|uniform) | `LAYER(784, 128, xavier)` |
| `QUANTIZE_W` | `(method)` | Binarize weights: `sign` → deterministic, `stochastic` → probabilistic | `QUANTIZE_W(sign)` |
| `QUANTIZE_A` | `(method)` | Binarize activations: `sign` → binary, `none` → identity (for output layer) | `QUANTIZE_A(sign)` |
| `STE_GRAD` | `(window)` | STE backward pass: `inf` → identity gradient, float → clip gradient to [-w,w] | `STE_GRAD(1.0)` |
| `LOSS` | `(type)` | Loss function: `ce` (cross-entropy), `mse` | `LOSS(ce)` |
| `UPDATE` | `(rule, lr, momentum?)` | Weight update: `sgd`, `adam`; optional momentum for SGD | `UPDATE(sgd, 0.1, 0.9)` |
| `SCHEDULE` | `(fn, param?)` | LR schedule: `const`, `cosine`, `step_decay(factor)` | `SCHEDULE(cosine)` |
| `CLIP_W` | `(min, max)` | Clamp latent weights to range after update | `CLIP_W(-1, 1)` |
| `FORWARD` | `(layers)` | Execute forward pass through layer sequence | `FORWARD([L1, L2])` |
| `TRAIN` | `(steps, batch)` | Training loop specification | `TRAIN(200, 50)` |

## 3. GRAMMAR

```
program     := layer_def+ training_config
layer_def   := ID "=" LAYER(int, int, init) ">>" QUANTIZE_W(method) ">>" QUANTIZE_A(method)
training_config := network loss_spec ste_spec optim_spec train_spec
network     := FORWARD([layer_id, ...])
loss_spec   := LOSS(ce | mse)
ste_spec    := STE_GRAD(float | inf)
optim_spec  := UPDATE(rule, float, float?) [">>" SCHEDULE(fn, param?)] [">>" CLIP_W(float, float)]
train_spec  := TRAIN(int, int)
```

**Key constraints:** Layer 1 uses `QUANTIZE_A(sign)`, Layer 2 uses `QUANTIZE_A(none)` for logit output. The `>>` operator chains configuration. All programs must specify exactly 2 layers with input 784 and final output 10.

## 5. EXAMPLE PROGRAMS

**Strategy A: Vanilla STE with Identity Gradient**
```
L1 = LAYER(784, 128, xavier) >> QUANTIZE_W(sign) >> QUANTIZE_A(sign)
L2 = LAYER(128, 10, xavier) >> QUANTIZE_W(sign) >> QUANTIZE_A(none)
FORWARD([L1, L2])
LOSS(ce)
STE_GRAD(inf)
UPDATE(sgd, 0.5) >> CLIP_W(-1, 1)
TRAIN(200, 50)
```

**Strategy B: Clipped STE with Cosine Schedule and Momentum**
```
L1 = LAYER(784, 128, kaiming) >> QUANTIZE_W(sign) >> QUANTIZE_A(sign)
L2 = LAYER(128, 10, kaiming) >> QUANTIZE_W(sign) >> QUANTIZE_A(none)
FORWARD([L1, L2])
LOSS(ce)
STE_GRAD(1.0)
UPDATE(sgd, 0.1, 0.9) >> SCHEDULE(cosine) >> CLIP_W(-1, 1)
TRAIN(200, 50)
```
