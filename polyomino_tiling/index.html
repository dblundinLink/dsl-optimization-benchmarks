# Polyomino Tiling DSL Specification

## 1. OVERVIEW

This DSL expresses strategies for placing 5 polyomino pieces (L-tromino, T-tetromino, I-tetromino, 2x2 square, L-tetromino) on a 4x4 board to achieve full coverage without overlap. The language constrains programs to valid tiling operations through explicit legality checks and structured backtracking, while enabling flexible strategy expression via piece ordering heuristics, position iteration, and optional reflections. Programs compile to a Python `tile()` function that returns placement tuples, abstracting board state management while exposing strategic decisions about search order and pruning.

## 2. OPERATORS

| Name | Args | Semantics | Example |
|------|------|-----------|---------|
| `PLACE` | `(piece, x, y, rot)` | Place piece at (x,y) with rotation; fails if illegal | `PLACE(L_TET, 0, 0, 90)` |
| `LEGAL` | `(piece, x, y, rot)` | Predicate: true if placement valid (in-bounds, no overlap) | `IF LEGAL(T_TET, 1, 1, 0)` |
| `BACKTRACK` | `()` | Undo last placement, try next option in search | `BACKTRACK()` |
| `REMAINING` | `()` | Returns set of uncovered squares | `WHILE REMAINING() != EMPTY` |
| `PRIORITY` | `(ordering)` | Set piece selection order: `SIZE_DESC`, `SIZE_ASC`, `CORNERS` | `PRIORITY(SIZE_DESC)` |
| `FOREACH` | `(var, collection)` | Iterate over PIECES, POSITIONS, or ROTATIONS | `FOREACH(p, PIECES)` |
| `REFLECT` | `(piece, axis)` | Mirror piece on H or V axis before placing | `REFLECT(L_TET, V)` |
| `SELECT` | `(strategy)` | Choose next item by strategy: `FIRST_FIT`, `MRV`, `RANDOM` | `SELECT(MRV)` |
| `CANDIDATES` | `(piece)` | Returns all valid (x, y, rot) tuples for piece | `FOREACH(pos, CANDIDATES(L_TET))` |
| `SOLVE` | `()` | Execute search with current strategy until complete/fail | `SOLVE()` |

**Pieces:** `L_TROM` (3 cells), `T_TET` (4 cells), `I_TET` (4 cells), `SQUARE` (4 cells), `L_TET` (4 cells)
**Rotations:** `0`, `90`, `180`, `270`

## 3. GRAMMAR

```
program     ::= setup* body SOLVE()
setup       ::= PRIORITY(ordering) | SELECT(strategy)
body        ::= statement+
statement   ::= placement | loop | conditional | BACKTRACK()
placement   ::= PLACE(piece, x, y, rot)
loop        ::= FOREACH(var, collection) DO statement+ END
              | WHILE predicate DO statement+ END
conditional ::= IF predicate THEN statement+ [ELSE statement+] END
predicate   ::= LEGAL(piece, x, y, rot) | REMAINING() != EMPTY
collection  ::= PIECES | ROTATIONS | CANDIDATES(piece) | REMAINING()
piece       ::= L_TROM | T_TET | I_TET | SQUARE | L_TET | REFLECT(piece, axis)
```

## 4. COMPILATION TARGET

The DSL compiles to a Python function with signature:

```python
def tile() -> list[tuple[int, int, int, int]]:
```
**Returns:** List of placement tuples `(piece_index, x, y, rotation)` where piece_index maps to: 0=L_TROM, 1=T_TET, 2=I_TET, 3=SQUARE, 4=L_TET. Coordinates are 0-indexed from top-left. Rotation is degrees (0/90/180/270). The function returns the best valid tiling found, maximizing `pieces_placed_correctly`.

## 5. EXAMPLE PROGRAMS

**Strategy A: Size-Descending DFS with Backtracking**
```
PRIORITY(SIZE_DESC)
WHILE REMAINING() != EMPTY DO
  FOREACH(p, PIECES) DO
    FOREACH(pos, CANDIDATES(p)) DO
      IF LEGAL(p, pos.x, pos.y, pos.rot) THEN
        PLACE(p, pos.x, pos.y, pos.rot)
      ELSE
        BACKTRACK()
      END
    END
  END
END
SOLVE()
```

**Strategy B: Corner-Anchored Fixed Placement**
```

PLACE(I_TET, 0, 2, 0)
PLACE(L_TET, 2, 0, 90)
PLACE(T_TET, 1, 2, 180)
PLACE(L_TROM, 3, 1, 270)
SOLVE()
```
