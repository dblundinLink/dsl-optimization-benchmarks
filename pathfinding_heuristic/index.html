# DSL Specification: A* Pathfinding Heuristic Composition Language

## 1. OVERVIEW

This DSL enables the composition of heuristic functions for A* pathfinding on 16×16 mazes. It provides distance primitives (manhattan, euclidean, chebyshev) as the foundation, augmented with maze-aware operators that capture local structure (wall density, openness, dead-ends) without expensive global computation. The design constrains heuristics to admissible-friendly compositions through weighted combinations, allowing optimization over continuous parameters while keeping evaluation O(r²) per cell where r is the maximum scan radius.

## 2. OPERATORS

| Name | Args | Semantics | Example |
|------|------|-----------|---------|
| `DIST` | `(type, weight)` | Distance to goal: `manhattan`\|`euclidean`\|`chebyshev`, scaled by weight | `DIST(manhattan, 1.0)` |
| `WALL_PENALTY` | `(radius, decay)` | Sum of 1/d^decay for each wall within radius of current cell | `WALL_PENALTY(2, 1.5)` |
| `OPENNESS` | `(radius)` | Count of free cells within radius, normalized to [0,1] | `OPENNESS(3)` |
| `DEAD_END` | `(threshold)` | Returns 1.0 if cell has ≤threshold open neighbors, else 0.0 | `DEAD_END(1)` |
| `GOAL_ALIGN` | `(weight)` | weight × cos(angle between cell-to-goal vector and best open direction) | `GOAL_ALIGN(0.2)` |
| `TIE_BREAK` | `(scale)` | Tiny perturbation: scale × cross-product of path vectors (breaks symmetry) | `TIE_BREAK(0.001)` |
| `CONST` | `(value)` | Constant float value | `CONST(0.5)` |
| `ADD` | `(a, b)` | Sum of two terms | `ADD(DIST(...), WALL_PENALTY(...))` |
| `MUL` | `(term, factor)` | Multiply term result by constant factor | `MUL(OPENNESS(2), -0.3)` |
| `MAX` | `(a, b)` | Maximum of two terms | `MAX(DIST(manhattan,1), DIST(chebyshev,1))` |
| `MIN` | `(a, b)` | Minimum of two terms | `MIN(a, b)` |
| `COMBINE` | `(t1, t2, ...)` | Sum of all nested terms (variadic ADD) | `COMBINE(t1, t2, t3)` |

## 3. GRAMMAR

```
heuristic  ::= expr
expr       ::= DIST(disttype, NUM) | WALL_PENALTY(INT, NUM) | OPENNESS(INT)
             | DEAD_END(INT) | GOAL_ALIGN(NUM) | TIE_BREAK(NUM) | CONST(NUM)
             | ADD(expr, expr) | MUL(expr, NUM) | MAX(expr, expr) | MIN(expr, expr)
             | COMBINE(expr, expr [, expr]*)
disttype   ::= manhattan | euclidean | chebyshev
NUM        ::= float literal (e.g., 1.0, 0.5, -0.2)
INT        ::= positive integer (e.g., 1, 2, 3)
```

**Syntax notes:**
- `COMBINE` accepts 2+ terms and sums them (syntactic sugar for nested `ADD`)
- `MUL` always takes a term and a constant (not two terms)
- Radius/threshold arguments are integers ≥1; weights/factors are floats
- Negative weights in `MUL` create bonuses (subtract from heuristic)

## 4. COMPILATION TARGET

The DSL compiles to a Python function with this signature:

```python
def heuristic(cell: tuple[int, int], goal: tuple[int, int], maze: list[list[int]]) -> float:
    """
    Args:
        cell: (row, col) of current position
        goal: (row, col) of target position  
        maze: 2D grid where 0=free, 1=wall
    Returns:
        Estimated cost from cell to goal (float, non-negative preferred)
    """
```

The maze is 16×16. Lower heuristic values indicate cells estimated closer to goal. For admissibility, the heuristic should not overestimate true shortest-path distance.

## 5. EXAMPLE PROGRAMS

**Anchor 1: Weighted Manhattan with Tie-Breaking**
```
COMBINE(
  DIST(manhattan, 1.0),
  TIE_BREAK(0.001)
)
```
*Pure distance heuristic with symmetry breaking to reduce node expansion on uniform-cost paths.*

**Anchor 2: Manhattan with Wall Awareness and Dead-End Penalty**
```
COMBINE(
  DIST(manhattan, 1.0),
  MUL(WALL_PENALTY(2, 1.0), 0.15),
  MUL(DEAD_END(1), 2.0),
  MUL(OPENNESS(2), -0.1)
)
```
*Adds penalties for wall-adjacent cells and dead-ends, with a small bonus for open areas, guiding search away from problematic maze regions.*
