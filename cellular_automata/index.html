# CellRule DSL Specification

## 1. OVERVIEW

CellRule is a domain-specific language for defining 2-state, 1-dimensional cellular automaton rules within the 256-rule Wolfram space. The DSL enables both direct rule specification and compositional exploration through modifiers, supporting strategies from explicit pattern tables to algebraic expressions. It compiles to a Python function that maps 3-neighbor configurations to output states, optimizing for visual complexity measured by compression ratio after 200 evolution steps.

## 2. OPERATORS

| Name | Args | Semantics | Example |
|------|------|-----------|---------|
| `WOLFRAM` | (n) | Load Wolfram rule number 0-255 as base | `WOLFRAM(110)` |
| `RULE` | (L,C,R -> N) | Define output N for neighborhood pattern (L,C,R) | `RULE(1,1,0 -> 1)` |
| `TOTALISTIC` | (sum -> N) | Output N when L+C+R equals sum (0-3) | `TOTALISTIC(2 -> 1)` |
| `XOR` | (a, b) | Output = a XOR b where a,b ∈ {L,C,R} | `XOR(L, R)` |
| `DEFAULT` | (N) | Fallback output for undefined patterns | `DEFAULT(0)` |
| `EXCEPTION` | (L,C,R -> N) | Override specific pattern after base rule | `EXCEPTION(1,1,1 -> 0)` |
| `SYMMETRY` | (mirror) | Enforce left-right symmetry (LCR = RCL) | `SYMMETRY(mirror)` |
| `INVERT` | () | Flip all outputs (0↔1) | `INVERT()` |
| `COMBINE` | (op, r1, r2) | Merge two rule numbers via AND/OR/XOR | `COMBINE(XOR, 30, 90)` |

## 3. GRAMMAR

```
<program>   ::= <base> <modifier>*
<base>      ::= WOLFRAM(<int>) | <explicit>+ | <totalistic>+ | <xor_expr>
<explicit>  ::= DEFAULT(<bit>) | RULE(<bit>,<bit>,<bit> -> <bit>)
<totalistic>::= TOTALISTIC(<sum> -> <bit>)
<xor_expr>  ::= XOR(<var>, <var>)
<modifier>  ::= SYMMETRY(mirror) | INVERT() | EXCEPTION(<bit>,<bit>,<bit> -> <bit>)
             | COMBINE(<op>, <int>, <int>)
<var>       ::= L | C | R
<bit>       ::= 0 | 1
<sum>       ::= 0 | 1 | 2 | 3
<int>       ::= 0..255
<op>        ::= AND | OR | XOR
```

**Evaluation order:** Base rule is established first, then modifiers apply in sequence. `EXCEPTION` overrides specific patterns. `SYMMETRY` enforces that mirrored patterns produce identical outputs. `INVERT` flips the final lookup table.

## 4. COMPILATION TARGET

The DSL compiles to a Python function with signature:

```python
def rule(left: int, center: int, right: int) -> int:
    """
    Args: left, center, right - each is 0 or 1 (neighbor states)
    Returns: 0 or 1 (next state for center cell)
    """
```

The compiler builds an 8-entry lookup table indexed by `(left << 2) | (center << 1) | right`, then returns the corresponding output bit.

## 5. EXAMPLE PROGRAMS

**Strategy 1: XOR-Based Chaotic Rule (Rule 90 variant)**
```
XOR(L, R)
EXCEPTION(1,1,1 -> 1)
EXCEPTION(0,0,0 -> 0)
```
*Rationale:* XOR rules produce fractal Sierpinski-like patterns; exceptions add asymmetry to increase complexity.

**Strategy 2: Modified Wolfram Rule with Symmetry**
```
WOLFRAM(110)
SYMMETRY(mirror)
EXCEPTION(1,1,1 -> 0)
EXCEPTION(0,0,0 -> 0)
INVERT()
```
*Rationale:* Starts from Turing-complete Rule 110, enforces visual balance via symmetry, prevents uniform regions with exceptions, then inverts to explore the complementary rule space.
