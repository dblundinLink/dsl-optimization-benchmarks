# Boids DSL Specification

## 1. OVERVIEW

This DSL defines declarative behavior rules for particle flocking (boids) simulations in 2D bounded space. Programs compose steering forces—separation, alignment, cohesion, and extensions—with explicit radii and strengths, compiling to a vectorized `update` function. The design constrains users to physically-motivated force compositions while enabling systematic parameter optimization for swarm coherence metrics. All operators are additive steering forces that blend according to specified weights, ensuring predictable emergent behavior without imperative control flow.

## 2. OPERATORS

| Name | Args | Semantics | Example |
|------|------|-----------|---------|
| `NEIGHBORS` | `(radius, max_count?)` | Limit neighbor queries to boids within radius, optionally capped | `NEIGHBORS(50, 10)` |
| `SEPARATION` | `(radius, strength)` | Steer away from neighbors within radius, inverse-distance weighted | `SEPARATION(20, 1.5)` |
| `ALIGNMENT` | `(radius, strength)` | Match average velocity of neighbors within radius | `ALIGNMENT(50, 1.0)` |
| `COHESION` | `(radius, strength)` | Steer toward center of mass of neighbors within radius | `COHESION(60, 0.8)` |
| `BOUNDARY` | `(mode, margin, strength)` | Handle edges; mode is `reflect`, `wrap`, or `repel` | `BOUNDARY(repel, 30, 2.0)` |
| `SPEED` | `(min, max)` | Clamp velocity magnitude to range | `SPEED(2, 6)` |
| `GOAL` | `(x, y, weight)` | Attract toward fixed position with given weight | `GOAL(400, 300, 0.5)` |
| `AVOID` | `(x, y, radius, strength)` | Repel from point when within radius | `AVOID(200, 200, 50, 3.0)` |
| `NOISE` | `(strength)` | Add random steering perturbation each step | `NOISE(0.1)` |
| `COMBINE` | `(mode)` | Force blending: `sum`, `average`, or `weighted` | `COMBINE(sum)` |
| `WHEN` | `(condition, operator)` | Conditional rule activation | `WHEN(step > 200, GOAL(...))` |
| `RING` | `(inner_r, outer_r, attract, repel)` | Mexican-hat: attract at outer, repel at inner | `RING(15, 50, 1.0, 2.0)` |

## 3. GRAMMAR

Programs follow this structure:
```
BOIDS {
  <config_operators>
  <behavior_operators>
}
```

**Config operators** (order-independent, at most one each): `NEIGHBORS`, `BOUNDARY`, `SPEED`, `COMBINE`

**Behavior operators** (one or more): `SEPARATION`, `ALIGNMENT`, `COHESION`, `GOAL`, `AVOID`, `NOISE`, `RING`, `WHEN`

**Conditions for WHEN**: `step > N`, `step < N`, `neighbors < N`, `neighbors > N`

Numeric arguments are floats or ints. Boundary modes are unquoted keywords: `reflect`, `wrap`, `repel`. Combine modes: `sum`, `average`, `weighted`.

## 4. COMPILATION TARGET

The DSL compiles to a Python function:

```python
def update(positions: np.ndarray, velocities: np.ndarray, step: int) -> tuple[np.ndarray, np.ndarray]:
    """
    Args:
        positions: shape (N, 2) - current boid positions
        velocities: shape (N, 2) - current boid velocities  
        step: current simulation step (0-499)
    Returns:
        new_positions: shape (N, 2)
        new_velocities: shape (N, 2)
    """
```

The world bounds are `(0, 0)` to `(800, 600)`. Initial positions and velocities are provided by the simulator. The function is called 500 times per evaluation.

## 5. EXAMPLE PROGRAMS

**Classic Reynolds Boids (balanced forces):**
```
BOIDS {
  NEIGHBORS(50, 10)
  SPEED(2, 5)
  BOUNDARY(repel, 40, 1.5)
  COMBINE(sum)
  SEPARATION(25, 1.5)
  ALIGNMENT(50, 1.0)
  COHESION(50, 1.0)
  NOISE(0.05)
}
```

**Tight Cohesion with Goal Seeking:**
```
BOIDS {
  NEIGHBORS(80, 15)
  SPEED(1.5, 4)
  BOUNDARY(repel, 50, 2.0)
  COMBINE(weighted)
  SEPARATION(15, 2.0)
  ALIGNMENT(60, 0.5)
  COHESION(80, 2.5)
  GOAL(400, 300, 0.3)
  WHEN(step > 300, NOISE(0.02))
}
```
